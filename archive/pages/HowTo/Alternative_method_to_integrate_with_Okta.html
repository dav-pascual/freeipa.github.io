Goals of this methodology¶

Okta support for OpenLDAP very reduced, it just allows you to delegate logins. When login succeeds a new user is imported int Okta. LDAP support doesn’t permit you to provision users in LDAP, activate/deactivate a user, push a password, and so on.

Following this Howto you will be able to:

  • provision users in FreeIPA from Okta

  • push passwords from Okta

  • disabling accounts from Okta

  • Desktop SSO using Kerberos and Ipsilon using as Desktop clients: Linux, MAC OSX or Windows.

Bear in mind that as a bridge between Okta an FreeIPA you will need a Windows Server that will be used just for provisioning/deprovisioning purposes
