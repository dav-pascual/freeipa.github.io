Overview¶

Ticket 3534 Remove the –selfsign option

In a future, we would like to support 2 flavors of certificate management in IPA:

  • IPA with pki-ca (dogtag) with either a self-signed certificate or with a certificate signed by external CA (–external-ca option)

  • IPA with no pki-ca installed with certificates signed and provided by an external CA.

Installation with –selfsign (selfsigned certificate managed in local NSS database on server) is rather troublesome and not even supported - it should be dropped.

Use Cases¶

  1. User tries passing the –selfsign option to ipa-server-install.

  2. The install fails as there is no such option.

  3. User upgrades a server that uses the self-signed CA

  4. The CA continues to work normally

Design¶

The –selfsign option to ipa-server-install will be removed.

Existing self-signed CAs should continue working for now, but the functionality is untested, and may be removed entirely in the near future.

Implementation¶

No impact, self-signed CAs are incapable of replication

Self-signed CAs should continue to work after upgrading to the new version. As before, they are neither tested nor supported.
